# HP 1630/1631A firmware — annotated disassemblies

One `.asm` per EPROM, generated by `f9dasm` driven by a per-ROM `.info` control
file (labels, code/data regions, FCC strings). Method: `6809_HEURISTICS.md` +
emulator coverage (`../listings/coverage.txt`) + string inventory
(`../strings/`) + semantic labels (`../annotations/`).

Regenerate any ROM:
`./tools/f9dasm -6809 -info asm/<ROM>.info -offset <base> roms/01630_<n>.bin > asm/<ROM>_annotated.asm`

| ROM (part) | chip/socket | role | base | entry structure |
|---|---|---|---|---|
| B0061 (80061) | 7 / U4H | **kernel** (reset, MMU, printf `$F54F`, self-test) | $C000 | reset/IRQ vectors @ $FFF0; `main_entry $F089` |
| B0055 (80055) | 1 / U3J | **disc / HP-IB** + config menu | $8000 | LBSR;RTS trampolines -> $814F/$8014/$80E0/$81AD |
| B0056 (80056) | 2 / U3I | **analog** subsystem self-test/UI | $8000 | LBSR;RTS trampolines -> $86E8/$8F08/$8095/$813E |
| B0057 (80057) | 3 / U3H | **state/timing config menu** | $8000 | LBSR;RTS trampolines -> $848E/$81D4/$83E8/$9410 |
| B0058 (80058) | 4 / U4K | **acquisition + listing** | $6000 | **JMP table**: $601B/$CB82/$CB61/$6630/$6586/$A27A/$FAA5/$7E96/$97D4 |
| B0059 (80059) | 5 / U4J | **measurement UI** (histogram/waveform/stats) | $8000 | code @ low half; uic_969F alloc/clear |
| B0060 (80060) | 6 / U4I | overlay (measurement) | $A000 | (code-direct) |
| B0054 (80054) | 0 / U3K | **prompt/error/status messages** + banner | $8000 | mostly data (message catalog) |

## Caveats
- **Banking / base**: each overlay is position-independent (PCR + cross-bank
  LBSR), and a single ROM can run different halves at different logical bases
  (e.g. B0058 low half @ $6000 via its JMP table, high-half acq code @ $5000 when
  `mmu_set_page` maps it). The `.asm` is disassembled at ONE base (above); absolute
  refs to other banks/bases show as their raw logical address. The **file offset**
  (`*XXXX:` minus base) is the bank-independent identifier; annotations key on it.
- Strings render as `FCC`; menu/form labels are LITERAL TEXT in display-list
  templates (see `../RE_NOTES.md`). Where code flow reaches a string, mark a
  blanket `data` region (done for menu-heavy ROMs) so FCC renders cleanly.
- B0056/B0058/B0060 use per-string FCC (code lives in their high halves), so a
  few template control-bytes may decode as instructions — cosmetic.

## Per-ROM semantic labels
`../annotations/B00{61,59,58,57,56}_labels.txt` (routines, I/O, strings).
`../strings/B00xx_strings.txt` = code-referenced strings (PCR) + menu literals.
